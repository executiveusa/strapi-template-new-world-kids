Steve Krug "Don't Make Me Think" — 20-Law Checklist
# Law Result Evidence
1 Don't make me think — obvious page purpose PASS Hero headline + proverb immediately communicates mission. No confusion about what the organization does.
2 Self-evident site sections PASS Programs, Timeline, Trust, Support sections are labeled clearly. No mystery meat navigation.
3 Clear visual hierarchy per page PARTIAL Hierarchy is strong in hero and programs. Studio section eyebrow uses gold (should be muted) making it compete with stats semantically.
4 Navigation is obvious and consistent PARTIAL 7 nav items is at the cognitive load ceiling. No active state indicator. Donate CTA hidden at mobile xs.
5 The home page answers: what is this place? PASS "Most kids graduate without ever learning the most important life skills. We fix that. Free. No exceptions." — answers in 2 sentences.
6 Search or clear content pathway PASS Single-page layout with section anchors in nav. No need for search on this scale.
7 Breadcrumbs / wayfinding for deep pages PASS Single-page site; wayfinding not applicable at this scale.
8 Conventions honored (logo top-left, nav top) PASS Standard header layout. Logo top-left. Donate CTA top-right. Consistent with web conventions.
9 Content is scannable (headers, bullets, short paras) PASS Section headers, pillar cards, and timeline chapters support scanning. No wall-of-text paragraphs.
10 Omit needless words PASS Copy is tight. No filler phrases detected. "We fix that. Free. No exceptions." is a master class in brevity.
11 Happy path is obvious PARTIAL Hero CTA leads to donation. But the Support section tier buttons barely look like buttons (transparent background). The path from interest to action has friction.
12 Forms and interactive elements are obvious PARTIAL Timeline cards look like content cards but are interactive (click opens modal). No visual affordance (no hover cursor, no arrow indicator). Violates Krug's "obvious" principle.
13 Error messages are helpful PASS No form error states to audit on this page — external donation platforms handle this.
14 Minimize choices (Hick's Law) PARTIAL 7 nav items + donate = 8 simultaneous choices in header. 3 support tiers is appropriate. Programs section correctly limited to 2 cards.
15 Feedback on interactive actions PARTIAL Studio CTA hover state is invisible (hover and base have identical colors). Non-featured support tier buttons barely change on hover.
16 Accessibility — can keyboard users navigate? FAIL Timeline cards have no tabIndex, role, or onKeyDown. Modal has no focus trap. No skip-to-main link. This is a WCAG 2.1 AA failure.
17 Tested with real users (or equivalent usability signals) FAIL All 3 testimonials flagged status: "needs-review" in siteData — real users' words are not shown, removing a critical trust signal.
18 Trust signals are prominent and verifiable PARTIAL EIN is shown. Trust PDFs are defined in code but not linked from the Trust section. IRS verification mention is plain text, not a hyperlink.
19 Site works when things go wrong (fragile deps) FAIL 6 timeline photos use Google Drive sharing URLs. If Google Drive access changes, all 6 timeline chapter images break simultaneously.
20 Mobile experience is not an afterthought PARTIAL Layouts reflow well. But Donate button is hidden sm:inline-flex — completely invisible at mobile XS, removing the primary conversion CTA from the most common screen size.
Result: 10 PASS  ·  7 PARTIAL  ·  3 FAIL

Priority 1 — Must Fix Before Approval
P1-001 · Copy Error · Factual
"Support all three programs →" — only 2 programs exist
Renders false information. Donors who count the cards will see the contradiction. Destroys credibility at the conversion moment.
ProgramsSection.tsx:106 — change "all three" to "both"
P1-002 · Copy Deviation · Canonical
Timeline headline: "living school" should be "living ecosystem"
Deviates from CANONICAL_COPY_v3. "School" only captures one pillar (education). "Ecosystem" captures the full food/water/energy/shelter scope and ties back to the organization's ecological mission.
TimelineSection.tsx:211 — change "a living school" to "a living ecosystem"
P1-003 · Accessibility · WCAG 2.1.1 + 2.1.2
Timeline cards and modal are keyboard-inaccessible
Interactive article cards have no tabIndex, role, or onKeyDown. The modal that opens has no focus trap. Keyboard-only and screen reader users cannot access any timeline content.
TimelineSection.tsx — add tabIndex={0}, role="button", onKeyDown to cards; add focus trap to modal
P1-004 · Color Violation · Design System
Studio section eyebrow uses gold token instead of eyebrow token
Gold is reserved for stats and prices. Using it for a section eyebrow dilutes the semantic signal and makes the design system's rules meaningless at a prominent position.
StudioSection.tsx:22 — replace --color-accent-gold with --color-eyebrow
P1-005 · Accessibility · WCAG 2.1.2
Donate button hidden at XS breakpoint — primary conversion CTA disappears on mobile
The most important action on the page (donate) is invisible to users on screens smaller than the sm breakpoint. Mobile is likely the majority traffic source for this demographic.
SiteHeader.tsx — change hidden sm:inline-flex to always visible, or add mobile-friendly donate path

P2-004 · Trust · Data Not Wired

trustDocuments defines 4 PDF links (Fiscal Sponsorship Letter, HSI Determination Letter, Federal Tax Exemption, Signed Agreement). TrustSection renders none of them. IRS verification is plain text, not a link.

P2-005 · Conversion · Missing Amount Params
Support tier CTA buttons don't pass amount to donation page
All 3 tier CTAs link to /donate without a query param (e.g. ?amount=25). The donation platform almost certainly supports pre-selecting an amount — a significant conversion optimization.
SupportSection.tsx — add amount query params to each tier's href
P2-006 · UX · Invisible Buttons
Non-featured support tier "Give" buttons barely visible
Using bg-[var(--color-border-subtle)] on a dark background makes the button nearly invisible. Swap to bg-[var(--color-surface)] with an explicit border, or use a visible action color.
SupportSection.tsx (~line 85)
P2-007 · Accessibility
Hamburger nav button missing aria-controls; modal missing role/aria-modal
Hamburger needs aria-controls="mobile-nav" + matching id on the panel. Timeline modal needs role="dialog", aria-modal="true", aria-label.
SiteHeader.tsx; TimelineSection.tsx modal element
P2-008 · Accessibility
No skip-to-main-content link
WCAG 2.4.1 requires a mechanism to bypass repeated navigation. Add a visually-hidden <a href="#main-content">Skip to main content</a> as the first focusable DOM element.
Layout or SiteHeader.tsx — add before the nav
